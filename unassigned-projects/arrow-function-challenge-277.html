ARROW FUNCTION CHALLENGE (277)

1. Standard Function Syntax

Description: This is the original and most formal way to declare functions (pre-ES6). It is verbose, requiring the function keyword, parentheses, and curly braces.
Pros: It binds its own this context, making it suitable for object methods and constructor functions.
Cons: The manual binding of this often causes bugs in callbacks and event handlers, requiring extra code (like .bind()) to fix.

a = 5
b = 4

a + b = ?

2. Arrow Function Rewrite

Description: Introduced in ES6, the arrow function is a concise syntax that drops the function keyword.
Pros: Excellent for callbacks, event handlers, and array methods (like map/filter) due to its lexical this (it inherits this from the surrounding scope). Single-line functions allow for **implicit returns**.
Cons: Cannot be used as object constructors and is not suitable for object methods where an independent this context is needed.

a = 5
b = 4

a + b = ?
